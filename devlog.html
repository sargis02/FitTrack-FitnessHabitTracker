Development Log

Phase 1: Activity Form + Display

Day 1

Built: The main HTML structure including header, navigation bar, and card-based layout.

Reason: I wanted to first create a clear structure before adding any functionality so that everything would be organised from the start.

Reflection: This helped me understand how to divide a web app into sections (form, stats, list, etc.). I also decided early to use a card layout because it makes the UI easier to read and expand later.

Phase 2: Delete Functionality

Day 2

Built: A form to add activities and a function to display them dynamically on the page.

Challenge: I realised that user input could break the layout if it included special characters.

Solution: I created an escapeHtml() function to safely display user input.

Reflection: This was my first time thinking about security and not just functionality. It made me realise that even simple apps need to handle user input carefully.
